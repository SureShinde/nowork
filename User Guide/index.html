How to install the theme?



Please do not install theme on live store direcly. First, install the theme on development environment then move it to the Production.

Prepare your magento

1. Refresh and Disable all the Magento cache

When configuring your store, whatever theme or anything else you should refresh all the Magento cache and then completely disable it. To get it done go to System -> Cache Management

2. Reindex magento data

After making any changes to magento data (products, categories) you should reindex the data. To get it done go to System -> Index Management

3. Backup Magento files and the store database

It saves lot's of your time if something goes wrong. To create backup go to System-> Tools-> Backups

4. Disable compilation mode

To get it done go to System -> Tools -> Compilation

5. Disable Merge JavaScript and CSS Files

To get it done go to System -> Configuration -> Developer

6. Log out from Magento admin panel

It must be done to refresh the magento access control system

Upload the theme files

  1. Using your favourite FTP client upload .zip file Theme Files/Theme_vX.X.zip from the downloaded package to the root directory of your magento
  2. Unzip Theme_v.X.X.zip

    However, .zip folder is contains the same folder structure as Magento have but it is built to serve the overriding on Magento's core file structure we didn't modified any core file of magento.
  3. Set correct file permissions for uploaded folders. More info about file permissions you can find here Magento WIKI: Filesystem permission
  4. Log in to magento admin panel

Install the theme

There are two ways how you can get the theme installed and configured

Using "Theme Default Setting" feature

This feature allows you to get your store installed and configured in a few clicks but we do not recommend using it on live stores because you can lose your homepage or/and static blocks. This feature is provided only for testing the theme on a test server. To install theme using this feature go to System -> Configuration -> Carty -> Theme Default Settings. there you can see two blocks Import CMS Blocks and Import CMS Pages where you have to first select both select box to yes which says Overwrite CMS Blocks and Overwrite CMS Pages then press save. It will override the All CMS block and CMS Pages with same identifier.

Manual Installation

This is the only right way of theme's installation if you prefer to install it on live store

  1. Go to System -> Configuration -> Design page of your admin panel and set the field "Current Package Name" as carty. Click Save Config button
  2. Create predefined static blocks. All used static blocks you can find under Sources/Static Content/Blocks, where <FILE NAME> - id of static block, <FILE CONTENT> - content of static block. Your static blocks may have any title
  3. Create predefined pages. All used pages you can find under Sources/Static Content/Pages, where <FILE NAME> - id of page, <FILE CONTENT> - content of page. Your pages may have any title. To create the homepage with same content as on demo please refer to Home Page chapter of this guide.
  4. Upload media folder from Sources/demo images/media to the root of your magento. Please notice that real images from preview are not included in the package due to licensing restrictions so are replaced with blank ones

Known problems with installation

If you get any problems with theme's installation first of all make sure that you have followed all steps indicated in "Prepare your magento" section.

404 Not Found when trying to open theme configuration page
Solution: It's pretty common issue. Just clear all the magento cache then logout from the admin panel and login again

"Package with this name does not exist and cannot be set" error when setting the package name
Solution 1. Make sure you have uploaded the theme files in the correct way. Note that folders structure of uploaded files must match to the structure of files in downloaded package.
Solution 2. Check if you have set correct file permissions

If theme installed correctly you will see dedicated carty settings under System -> configuration -> carty.

  • Menu underneath it you will find Default Menu Settings where you can see two option

    1. Display Breakpoint Before Banner If yes then you will be prompted for adding a breakpoint in px.

  • Category Page underneath it you will see Category Page Settings.
    1. Products per row This will work if page width is less than or equal to 1024. Due to fluid width : Number of product increases to set accordingly if page width is greater than 1024..
    2. Display Category Sidebar You can select it whether to display it on left side or right side

  • Product View Page underneath it you will find multiple settings which are dedicated to only product view page.

      1. Keep Image Aspect RatioSet to 'Yes' will take image height automatically based on image width itherwise it display with default size.

    1. Sidebar Block Settings
      1. Disable Brand Block on Left SidebarSet No to display static block on left side
      2. Disable Bestseller Block on Left SidebarSet No to display static block on left side

Sidebar



All blocks you can see in the sidebar of your store are default magento ones. If you have installed your store from scratch you will need to do some actions to get your store working.

If you are not very familiar to magento we strongly recommend you to install magento sample data.

Built-in Sidebar Blocks

  • Categories - block consists of categories' list. This block is based on header navigation but can't display any static content. Please refer to "MegaMenu" section of this guide to learn how to create categories correct way.
  • Shop By - this block consists of several product filters. To enable this block you may navigate to Catalog -> Manage Categories, click on a category you need and go to "Display Settings" Tab. Find "Is Anchor" option there and set it to "Yes"
  • Compare Products - to see this block add some products to comparison list by clicking button Add to compare on frontend.
  • Compared Products - it will appear after you compared some products.
  • Poll - it will appear after you configured it there CMS -> Polls.
  • Wishlist - this block appears when you have added some products in your wishlist. You can configure this feature there System -> Configuration -> Customers -> Wishlist
  • Popular tags - this block is hidden if your products haven't any tags. Add some tags being logged as customer on frontend (in that case you also will need to approve that tags in admin Catalog-> Tags-> Pending tags or go to Catalog -> Tags -> All Tags in admin and add some tags by clicking on button Add new Tag. To disable this feature completely go to System -> Configuration -> Advanced tab and set "Mage_Tag" to "Disable"
  • My orders - every customer that have already purchased some of your products will see that sidebar block with list of recently purchased products on frontend.
  • PayPal Logo - this block is appeared by default, to disable it go to System -> Configuration -> PayPal tab -> Frontend Experience panel. Set "PayPal Product Logo" to "No logo"
  • Newsletter - (if set in sidebar in this theme). To disable this functionality completely go to System → Configuration → Advanced tab and set "Mage_Newsletter" to "Disable"

Some of these blocks you can disable using our theme options extension here Meigee -> Theme Options -> Sidebar Options. Please be aware on that page you can only hide blocks from frontend but they won't be disabled at all.

Customization



Magento uses parent/child theming concept so our theme is based on default magento theme. It allows to customize any parts of a theme with ease.

You should never modify magento and our theme default template files. This may cause lot's of problems in updating magento core or our theme to the latest version.

How to customize the theme

There are two correct ways how you should customize our theme. Choose it based on your needs

  1. You need to customize css rules only. In this case you should write your own css rules in skin/frontend/indigo/default/css/custom.css
    Since this file is loaded after any other css files all css rules written in it will override any others
  2. You need to customize .js or .phtml files. In this case the only correct way is creating your own subtheme.

How to create subthemes

Creating a subtheme is a correct way of customization our theme. It allows you to update our theme to the latest version with ease. All you will need is just upload newest version of the theme implement changes in overridden files in your own subtheme. You can create as many subthemes as you want. You can create specific subthemes for categories or cms pages, etc.

Default theme folders structure.
app/design/frontend/indigo/default/
Template files are separated into following folders:
layout - contains XML files which define pages structure
template - contains template .phtml files
locale - contains CSV files with translations

skin/frontend/indigo/default/
Template files are separated into following folders:
css - contains CSS files
images - contains images
js - directory contains JavaScript files

Overriding of our extensions is not recommended and this guide doesn't cover it. You may have strong php/Zend knowledges to customize any extensions including ours.

To create your own subtheme you need to create two following folders:
app/design/frontend/indigo/subtheme/
skin/frontend/indigo/subtheme/

Then you need to copy files which you want to customize into that subtheme keeping folders structure. For example if you need to customize header app/design/frontend/indigo/default/template/page/html/header.phtml you need to copy it to app/design/frontend/indigo/subtheme/template/page/html/header.phtml

How to activate subtheme

After creating subtheme the way described above you need activate it via the magento admin panel. You can activate different themes for a whole store, categories or pages

To activate your own subtheme for a whole store go to System -> Configuration -> Design and fill needed fields like this:


To activate your own subtheme for a specific category go to System -> Manage Categories , click on a category you want to change theme for, go to the tab "Custom Design" and choose your own subtheme under indigo group:

About Third Party Extensions



There is the only one theme which is fully compatible with any extensions. Default magento theme.
We tried to make this theme as much compatible with any extensions as possible but it's impossible to get it done for sure due to huge amount of available extensions. So please notice this theme may not be compatible with some extensions.

Why the theme can't be compatible with any extensions?

The answer is pretty simple: If this theme and an extension use the same template files - they aren't compatible

Solution?

In most of cases the solution is pretty easy too:
Move template files of an extension to the theme folder. This theme is an interface, like base or default one, so if an extension has folder under base/default/ or default/default/ move all of them to indigo/default/

But sometimes you also need to edit some template files of an extension or/and of the theme ones to get it working. We strongly do not recommend doing this if you are not familiar to magento, hire professional developers to get it done instead. Please refer to Custom Development chapter of this guide for more information.

Authors of the 3-rd party extensions should provide detailed instruction about how to integrate the extension with custom themes because default magento theme is used only on magento demo, on real stores people use custom themes for sure.

Blog Extension

  3. Go to blog extension page to get extension link and then install it

If your blog page isn't shown in browser or seems broken make sure you have unziped extension files correct way. All folders and files of this extension must be in appropriate directories of your magento. If you see aw_blog folder in the root of your magento you uploaded/unziped it wrong way
