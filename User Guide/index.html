Package Structure



This package is organized as follows:
/Changes - contains .diff files where we describe all changes we have altered in code. All these changes must be implemented by yourself. Please refer to customization chapter of this guide to learn how to customize this theme in the correct way which allows you implement further updates with ease
/Licensing - contains license files
/Sources
-- demo images - contains media folder which must be uploaded to the root directory of your magento only if you use our "One Click Installation" feature

-- psd - contains layered .psd files
-- Static Content - contains static content used on the demo of the theme
-- -- Blocks - contains static blocks

/User Guide - you are reading it at the moment
/Online FAQ.url - click it twice to open our online FAQ in your browser

Template Installation



We strongly recommend you to install the theme on a test store first and only then move it to the live store

Prepare your magento

1. Refresh and Disable all the Magento cache

When configuring your store, whatever theme or anything else you should refresh all the Magento cache and then completely disable it. To get it done go to System -> Cache Management

2. Reindex magento data

After making any changes to magento data (products, categories) you should reindex the data. To get it done go to System -> Index Management

3. Backup Magento files and the store database

It saves lot's of your time if something goes wrong. To create backup go to System-> Tools-> Backups

4. Disable compilation mode

To get it done go to System -> Tools -> Compilation

5. Disable Merge JavaScript and CSS Files

To get it done go to System -> Configuration -> Developer

6. Log out from Magento admin panel

It must be done to refresh the magento access control system

Upload the theme files

  1. Using your favourite FTP client upload .zip file Theme Files/Theme_vX.X.zip from the downloaded package to the root directory of your magento
  2. Unzip Theme_v.X.X.zip

    Despite the fact that the .zip file contains the same folders as magento none of the magento core file will be overridden. It was made with the aim to make upload process easier
  3. Version of the magento the theme was created for, you can find on the top of this guide; if you want to use the theme for the different version you should upload theme folders from Theme Files/Theme_vX.X and then upload patch files located in Theme Files/patch_for_magento_community_1.x.x.x for specific version of magento
  4. Set correct file permissions for uploaded folders. More info about file permissions you can find here Magento WIKI: Filesystem permission
  5. Log in to magento admin panel

Install the theme

There are two ways how you can get the theme installed and configured

Using "1 Click Installation" feature

This feature allows you to get your store installed and configured in a few clicks but we do not recommend using it on live stores because you can lose your homepage or/and static blocks. This feature is provided only for testing the theme on a test server. To install theme using this feature go to Meigee -> Activate Theme. In that menu you can mark 2 checkbox in order all static blocks and pages were created automatically and choose store for which theme should be applied. Be aware that if you already created some static blocks or home page with your own content they will be replaced with theme static content in case ID of your blocks and page will match predefined static blocks and pages.

Manual Installation

This is the only right way of theme's installation if you prefer to install it on live store

  1. Go to System -> Configuration -> Design page of your admin panel and set the field "Current Package Name" as indigo. Click Save Config button
  2. Create predefined static blocks. All used static blocks you can find under Sources/Static Content/Blocks, where <FILE NAME> - id of static block, <FILE CONTENT> - content of static block. Your static blocks may have any title
  3. Create predefined pages. All used pages you can find under Sources/Static Content/Pages, where <FILE NAME> - id of page, <FILE CONTENT> - content of page. Your pages may have any title. To create the homepage with same content as on demo please refer to Home Page chapter of this guide.
  4. Upload media folder from Sources/demo images to the root of your magento. Please notice that real images from preview are not included in the package due to licensing restrictions so are replaced with blank ones

Known problems with installation

If you get any problems with theme's installation first of all make sure that you have followed all steps indicated in "Prepare your magento" section.

404 Not Found when trying to open theme configuration page
Solution: It's pretty common issue. Just clear all the magento cache then logout from the admin panel and login again

"Package with this name does not exist and cannot be set" error when setting the package name
Solution 1. Make sure you have uploaded the theme files in the correct way. Note that folders structure of uploaded files must match to the structure of files in downloaded package.
Solution 2. Check if you have set correct file permissions

Header Slider



This theme is equipped with header images slider based on iosslider. Despite of the fact that this jQuery plugin is chargeable you can use it absolutely for free with this theme.

All available slider options you can find there Meigee -> Theme Options vX.X -> Header Slider

This slider is based on static blocks which are merged into slider automatically. All demo slides you can find under Sources/Static Content/Blocks, they called indigo_slide_X.html for wide slider and indigo_boxed_slide_X.html for boxed one.

To create header slider with your own content follow the instructions below:

  1. Upload your slider images on your server, using your favourite FTP client
  2. Create Static block for each slider image and fill it with content: add slider image to static block by clicking button "Insert Image" at the top of static block content field. Then add your promo text or other content to the slide and save changes. While editing the static blocks for header slider make sure that WYSIWYG editor is disabled. If it is available your slider images' code will be wrapped with tag p and the slider will not be displayed correctly. Be aware that for every slider image you have to create separate Static Block
  3. When you created all needed static blocks for header slider go to Meigee-> Theme Options-> Header slider and activate header slider by setting Yes in Enable dropdown menu
  4. Select the slider type (Boxed or Wide) and in field "Slides" indicate comma separated IDs of previously created static blocks with slider content
  5. Configure other slider options (slider timer, auto sliding ...) and save changes

Known problems with header slider

Slider images aren't displayed correctly

Solution 1. Disable WYSIWYG editor theme System -> Configuration -> Content Management
Solution 2. Check if your images aren't wrapped with html tag <p> like this <p><img ... /></p>. Images must be put without that tags
Solution 3. One of your slider images has another proportion than all other header slider images. All of header slider images should have same proportions

  2. Make sure that "Manufacturer" attribute is in "General" group:


    If you do not have such attribute you have to create it in admin. More information about attributes you can find in magento guide
  3. Go to: Catalog -> Attributes -> Manage Attributes click on "Manufacturer" from attribute list and go to the "Manage Label / Options" tab
  4. Make "Manufacturer" attribute options looking the same as on a following screenshot:

  5. Fill in option fields with your brand names and save the attribute:

  6. Create logo images of your brands with the same names as you set in the previous step. If you use different names for each store view you have (English, German, Spanish, etc...) you need create images with all of those names. We recommend to create logo images with same proportions.

Next widgets are available in Indigo theme:

  • New products Extended - shows products which are marked as new products
  • On Sale products Extended - shows products which are marked as products with special prices
All product widgets has 3 displaying modes: list, grid, slider. Grid and list has option that allows to set amount of visible products. Slider mode has additional options such as "amount of visible products" and auto slide feature. Widget id is auto generated field, so there is no need to fill it with data.

  • Latest tweets - shows recent tweets of your tweeter account
  • Social links - adds links of your favourite social networking websites. If you want to add more social network icons than are included in theme (i.e. tumblr link) you have to make next several changes. Go to Awesome icon website to the Brand section and copy the icon name (i.e. tumblr). Then go to app/code/local/Meigee/MeigeewidgetsIndigo/etc/widget.xml find string with value "</instagram>" and insert after it next code:

    <your_icon_name translate="label"> 
    	<label>Your_icon_name</label>
    	<type>text</type>
    	<visible>1</visible>
    	<sort_order>11</sort_order>
    	<description>Example: http://your_social_network_website_link/ (Leave empty to hide)</description>	
    </your_icon_name>
    Save changes in file. Next step is to go to app/design/frontend/indigo/default/template/meigee/meigeewidgetsindigo/sociallinks.phtml and before "echo '</ul>';" insert next code snippet:

    if (!empty($html['your_icon_name'])) echo '<li><a class="your_icon_name" href="'.$html['your_icon_name'].'"><i class="fa fa-your_icon_name"></i></a></li>';
    After all changes were saved, flush all the cache and the field for new icon will appear in widget options.
  • Brands Showcase - shows logo images of selected manufacturers on any cms page. In this widget options you can choose what brands you want to show, display mode of showcase (slider or simple list), select format of your brands logo images and make that logos click-able. Be aware when you choose "slider" option, it will work only if the sum width of your brand images is greater than 1252px, otherwise slider won't run.

    • Brand logo images aren't displayed after adding them via widget

    Solution 1. Check if brand logo image is created and exist in media/brands folder of your magento
    Solution 2. Check your brand logo image file name and name of brand in admin (Catalog -> Attributes -> Manage Attributes -> Manage Label / Options). They must match.
    Solution 3. Check if format of files matches the format of file. It must match. All brand logo images must have same format

Also the space before and after those dividers can be changed by applying another class. All classes that can be used for dividers are described below (just replace highlighted text in above divider code with needed value):

  • invisible - will make the divider invisible
  • solid - will make divider solid
  • dotted - will make divider dotted
  • dashed - will make divider dashed
  • indent-9 - will add .75em space before and after divider
  • indent-12 - will add 1em space before and after divider
  • indent-18 - will add 1.5em space before and after divider
  • indent-24 - will add 2em space before and after divider
  • indent-28 - will add 2.33em space before and after divider
  • indent-32 - will add 2.5em space before and after divider
  • indent-36 - will add 3em space before and after divider
  • indent-40 - will add 3.33em space before and after divider
  • indent-46 - will add 3.88em space before and after divider
  • indent-52 - will add 4.33em space before and after divider
  • indent-56 - will add 4.66em space before and after divider
  • indent-60 - will add 5em space before and after divider


Notice: Since magento default WYSIWYG editor breaks any HTML tags we recommend you to disable it before changing the content. To disable it go System -> Configuration -> General -> Content Management (very bottom of the list) -> Enable WYSIWYG Editor -> Disable by Default (or Disable Completely) and click "Save Config". Otherwise your content will be broken after any save action.

Known problems with a product page

Product options block is displayed below the product image

Solution. This is standard magento behaviour, to display this block on the right of the product image navigate to a product edit page -> “Design” vertical tab and set “Product Info Column” in “Display Product Options In” field.

Brand logo is not displaying, while the option of showing brand logo is enabled.

Solution 1. Check if brand logo image is created and exist in media/brands folder of your magento
Solution 2. Check your brand logo image file name and name of brand in admin (Catalog -> Attributes -> Manage Attributes -> Manage Label / Options). They must match
Solution 3. Check if format of files matches the format of file. It must match
Solution 4. Assign product to the manufacturer on product edit page

If you do not see you related products in sidebar of product page whereas the fact that the Related block position is set as Sidebar please check the points below:

Solution 1. Related products should be assigned to that product Product edit page -> Related products
Solution 2. The reindex and flushing of cache should be done after assigning of related products. This is default magento behaviour: after applying any changes to the products or category you have to reindex all the data and flush the cache in order to see the changes on fronend immediately
Solution 3. The Product page sidebar position Meigee -> Theme Options -> Product Page Options should be chosen as Left or Right
Solution 4. Meigee Product options that concern related product block position and product sidebar location that are on product edit page should use Config settings if not the combination should be same as above
Notice: Theme design is developed for using with 1-column structure of the product page layout only. Sidebar with your own content on the product page you can add via theme options. Setting of 2- or 3-columns structure of product page layout by magento design options will require changing of theme code by yourself, since the product page layout will be broken.

Built-in Sidebar Blocks

  • Categories - block consists of categories' list. This block is based on header navigation but can't display any static content. Please refer to "MegaMenu" section of this guide to learn how to create categories correct way.
  • Shop By - this block consists of several product filters. To enable this block you may navigate to Catalog -> Manage Categories, click on a category you need and go to "Display Settings" Tab. Find "Is Anchor" option there and set it to "Yes"

  • Compared Products - it will appear after you compared some products.
  • Poll - it will appear after you configured it there CMS -> Polls.
  • Wishlist - this block appears when you have added some products in your wishlist. You can configure this feature there System -> Configuration -> Customers -> Wishlist
  • Popular tags - this block is hidden if your products haven't any tags. Add some tags being logged as customer on frontend (in that case you also will need to approve that tags in admin Catalog-> Tags-> Pending tags or go to Catalog -> Tags -> All Tags in admin and add some tags by clicking on button Add new Tag. Customization



Magento uses parent/child theming concept so our theme is based on default magento theme. It allows to customize any parts of a theme with ease.

You should never modify magento and our theme default template files. This may cause lot's of problems in updating magento core or our theme to the latest version.

How to customize the theme

There are two correct ways how you should customize our theme. Choose it based on your needs

  1. You need to customize css rules only. In this case you should write your own css rules in skin/frontend/indigo/default/css/custom.css
    Since this file is loaded after any other css files all css rules written in it will override any others
  2. You need to customize .js or .phtml files. In this case the only correct way is creating your own subtheme.

How to create subthemes

Creating a subtheme is a correct way of customization our theme. It allows you to update our theme to the latest version with ease. All you will need is just upload newest version of the theme implement changes in overridden files in your own subtheme. You can create as many subthemes as you want. You can create specific subthemes for categories or cms pages, etc.

Default theme folders structure.
app/design/frontend/indigo/default/
Template files are separated into following folders:
layout - contains XML files which define pages structure
template - contains template .phtml files

About Third Party Extensions



There is the only one theme which is fully compatible with any extensions. Default magento theme.
We tried to make this theme as much compatible with any extensions as possible but it's impossible to get it done for sure due to huge amount of available extensions. So please notice this theme may not be compatible with some extensions.

Why the theme can't be compatible with any extensions?

The answer is pretty simple: If this theme and an extension use the same template files - they aren't compatible

Solution?

In most of cases the solution is pretty easy too:
Move template files of an extension to the theme folder. This theme is an interface, like base or default one, so if an extension has folder under base/default/ or default/default/ move all of them to indigo/default/

But sometimes you also need to edit some template files of an extension or/and of the theme ones to get it working. We strongly do not recommend doing this if you are not familiar to magento, hire professional developers to get it done instead. Please refer to Custom Development chapter of this guide for more information.

Authors of the 3-rd party extensions should provide detailed instruction about how to integrate the extension with custom themes because default magento theme is used only on magento demo, on real stores people use custom themes for sure.

Below we tried to describe in a few words options of that extensions, the more detailed information about options you can get by reading the extensions promo pages or by addressing questions directly to developers of extensions.

IWD one page checkout has next options:
  1. General here you can activate custom onepage checkout, entitle that page, choose default payment method, set default country and decide if you want to show shipping address form, newsletter block and comment field on one page checkout page. Same here you can let your customers make orders without being loged in
  2. GeoIP function allows your customer not to fill such fields as country and city on onepage checkout, that fields will be auto filled based on IP of customer. Here you can activate that feature and indicate files which helps to realize that feature. GeoIP files you can download directly from magento admin by clicking on button "Download" below the filename fields
  3. Adress Form Fields submenu allows you to choose 1 of 3 statuses (hidden, optional and required) for address form fields
  4. Terms and Conditions this option will show Terms and Conditions on onepage checkout. You can show only agreement checkbox by choosing minimal output type of Terms and Conditions or display full text of Terms and Conditions by choosing standard output type. Besides activating of Terms and Conditions here you also have to create Terms and Conditions information in magento admin panel under Sales -> Terms and Conditions menu
Magehouse price slider consists of next submenus:
  1. Settings let you activate price slider. Dropdown lists "Include Jquery" and "Include Jquery UI" should be leaved with default values in order store with installed theme works correctly. By default "Include Jquery" should be set as "No" and "Include Jquery UI" as "Yes"
  2. Price Slider Configuration here you find options that allow to customize your price slider by applying custom colors and other styles
  3. Ajax Configuration in this submenu ajax options are gathered all together. Ajax for toolbar should be turned off since there is another Ajax extension included in theme which is used for that. All other options are self-explanatory and don't require detailed description
VS Ajax has next functional options: Ajax for add to wishlist/compare button - this options can be applied to the category and/or for product page, Ajax for add to cart button - option that enables ajax add to cart button on product page as well on category page and quick view on category page. Last option of this extension is Ajax for toolbar/paging that has only 1 option of activating. Keep in mind that quick view button is a part of one option (Ajax for add to cart button) so if you want to get rid of quick view button ajax add to cart button on category page will also be disabled.

Blog Extension



This theme includes blog extension developed by aheadWorks. This extension isn't installed by default so you have to follow a few next steps to get it working.

How to install

  1. Refresh and disable all magento cache
  2. Please read What is a magento connect extension key
  3. Go to blog extension page to get extension link and then install it
  4. Upload aw_blog.zip to the root directory of your magento using your favorite FTP client. This .zip you can find in Sources -> Extensions folder of downloaded package
  5. Unzip it
  6. Refresh magento admin page in your bowser
  7. Log out from magento admin panel and log in again

That's it! Your blog installed. Now you can add some posts to your blog play with available options

If your blog page isn't shown in browser or seems broken make sure you have unziped extension files correct way. All folders and files of this extension must be in appropriate directories of your magento. If you see aw_blog folder in the root of your magento you uploaded/unziped it wrong way

How to configure

Since we didn't devoped this extension we do not provide support for it, so please refer to the extension author with any questions you will have.

Since in our theme is used only left sidebar please go to Blog -> Settings -> Blog Settings Tab and set "Page layout" option to "2 column with left bar"

Theme Updates



First of all make sure you have flagged this option on your downloads page to get notified by email if the theme is updated

After getting an email with note that the theme is updated you need to download the latest version on your downloads page by yourself. Newest theme always comes with folder "Changes" where we put .diff files with all changes we made.

If you have customized the theme correct way which is described in Customization section of this guide all you need is follow Installation steps once again and then implement changes described in .diff file into your subtheme files if they were changed.

Otherwise you will need to read line by line .diff file and make the same changes in all affected files. Sometimes .diff files are really huge
